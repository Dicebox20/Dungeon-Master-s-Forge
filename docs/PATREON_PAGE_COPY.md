# Dungeon Master's Forge Patreon Page Copy

Updated: 2026-07-02

This file is draft marketing and support copy for the Patreon launch page.

It is meant to stay honest to the current release. It should not promise Patreon entitlement features before they exist.

## Short Headline

Dungeon Master's Forge turns plain-English D&D magic item ideas into Foundry-ready DND5e items.

## One-Sentence Pitch

Describe the item you want, review the result, and create a Foundry-ready item with mechanics, activities, charges, effects, and advanced automation support where the current tier allows it.

## Short About Section

Dungeon Master's Forge is a Foundry VTT toolkit for Dungeon Masters who want faster item creation without giving up control.

It helps turn natural-language item requests into DND5e item specifications you can review before anything is created in your world. The current release supports a local rules compiler, a Bring Your Own API workflow, Foundry-first item generation, and a growing library of proven item patterns.

## What Works Right Now

- Foundry VTT v14 and DND5e 5.3.3 support
- split Description and Result workflow
- review-before-create item generation
- weapons, armor, wondrous items, consumables, staffs, and more
- charges, recharge, SRD spell-linked items, summoning, enchantment, and multi-activity items
- Foundry-aware advanced support for DAE, Item Macro, and Midi-QOL-compatible output where supported
- hosted Free Forge generation with 20 requests per calendar month
- optional Bring Your Own API workflow for private or independent usage

## What This Is Best For

- Dungeon Masters who build a lot of homebrew items
- GMs who want Foundry-ready mechanics faster
- world builders who want flavor-rich items without hand-entering every field
- advanced Foundry users who want a portable baseline first and deeper automation where available

## Important Current Limitation

Free Forge is live for invited testing, but the stable public manifest and permanent project hostname have not completed their final promotion gate. Patreon accounts are not connected to the module, and the Founding Patron tier does not unlock additional product features at launch.

## Recommended Launch Tier Shape

Launch with two public-facing tiers:

- **Apprentice (Free):** the actual product access tier
- **Founding Patron:** a support-first early-backer tier

This keeps the launch honest. People can install and use the module for free, while supporters can back the project without being promised a hosted entitlement system that is not live yet.

## Honest "Not Yet" Section

Not live yet:

- Patreon entitlement enforcement inside the product
- managed user accounts
- full realm-linked scene package delivery
- automatic premium image generation as a standard feature

## Tier Framing Paragraph

The tiers do not limit creativity. They scale with mechanical complexity, automation depth, monthly usage expectations, and premium Foundry integrations.

If you want a wild cursed relic, a joke item, a legendary artifact, or a dramatic homebrew weapon, you can still imagine it at any tier. The higher tiers are about how deeply the Forge helps automate it.

## Suggested Public Tier Summary

- **Apprentice (Free):** install the module, use 20 hosted Free Forge generation requests per calendar month, or connect a compatible private service
- **Founding Patron:** **Thank you for your support.** This tier helps fund continued development without gating launch-day features

For launch-day messaging, keep the deeper multi-tier automation ladder as future planning rather than a public promise.
