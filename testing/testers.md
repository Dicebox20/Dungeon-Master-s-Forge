## Current Build

The current public tester build is **2.23.1-test.62**.

Install or update it from Foundry's **Add-on Modules** screen by using this manifest URL:

`https://raw.githubusercontent.com/Dicebox20/Dungeon-Master-s-Forge/dm_forge/tester/testing/module.json`

Keep the original prompt unchanged while reviewing a result. If a result is close but a mechanic is wrong or missing, click **Retry**, enter only the correction in **What should it have done?**, check **I understand**, and click **Send repair request** once. The repaired result returns for fresh review and approval; it does not create or execute anything automatically.

Use the [feature menu](DMF_FEATURE_MENU_FOR_TESTERS.md) to choose item features and automation behaviors to test. For a guided automation run, use the [one-at-a-time automation sweep and questionnaires](AUTOMATION_ONE_AT_A_TIME_SWEEP_2026-07-23.md).

For a broader first session, use the [ten-prompt tester sweep](TESTER_PROMPT_SWEEP_2026-07-23.md), which moves from simple items to mixed production automation without requiring deferred features.

## How To Help

Use the tester build, try prompts in your own words, and check the item in Foundry before you approve it. If something looks wrong, tell us what you expected and what actually happened. Console errors, exported item data, and screenshots are useful when they do not include private information or API keys.

For the full installation steps, limits, prompt suggestions, and reporting guidance, see the [tester guide](README.md).

Newly accepted testers can start with the [welcome message and first-session checklist](NEW_TESTER_WELCOME_DRAFT.md).
